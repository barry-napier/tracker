When owed: the agent-authored asset must exist (`demo/demo.spec.ts` for `ui`, `demo/demo.sh` for `api`) — checked before booting so a demo-less run fails fast without spawning; then the recorder boots via `PreviewManager.restart` (always fresh code; lifecycle transitions audit as actor `agent` — upsertPreview grew an actor), waits out readiness, records, and stops the preview. `ui` runs the worktree's own Playwright (`npx --no-install` — no install, missing Playwright fails honestly) against an orchestrator-authored config forcing `baseURL` + `video: "on"`, then persists the newest `.webm`; `api` runs the script with `$BASE_URL`/`$PORT` and persists the transcript (`ArtifactStore.persistContent`, kind `demo`, stamped with the worktree HEAD). `demo-fresh` mirrors pr-fresh: the ticket's newest demo artifact (any run — a stale survivor must be judged) must be recorded at the branch tip, SHA-compared; no artifact fails with the recorder's own reason (boot failure carries the log tail). Wizard: the ui walkthrough plays `demo.webm` via `<video>` from the artifact route (`video/webm` content type added); the api transcript panel now shows real recordings; docs step excludes demo-kind artifacts.
