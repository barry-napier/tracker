# GitHub device flow is identity; secrets encrypt behind a cipher seam

The app gates launch on a GitHub sign-in via OAuth device flow — a public client id in source, no client secret, the renderer never touching the token. The flow runs in the Hono server (which lives in Electron main): the device code stays server-side in memory keyed by an opaque session id, the renderer drives polling cadence, and the resulting token is encrypted with an injected `SecretCipher` and stored as one row in SQLite. Electron main injects a `safeStorage` (keychain) cipher; outside Electron the plaintext fallback applies and the row records `token_plaintext_fallback = 1` — honest, never silent. Scopes are `read:user user:email` only: this token is identity (avatar, login, email for the settings surface), while every repo operation stays on the `gh` CLI (`GitHubPort`), whose credentials remain the operator's own. Sign-out deletes the row locally; revoking the grant server-side would need a client secret a public device-flow client deliberately lacks. `TRACKER_NO_AUTH=1` disarms the gate for tests and headless dev.
