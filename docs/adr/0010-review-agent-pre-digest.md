# Review agent: pre-digest the diff, never the verdict

The gate battery is mechanical and the wizard used to start from raw evidence: the reviewer's first minutes went to reconstructing what changed and where the risk lives. TRK-3 inserts a review-agent session between the battery going green and the ticket entering Human Review: `ReviewAgent` (review-agent.ts) reads the finished diff in the run's worktree and writes `kb/review-digest.json` — an annotated walkthrough in reading order, risk callouts with severity, and an AC-to-code mapping. The orchestrator lints the structure (`lintReviewDigest`), persists it as Run evidence (kind `review-digest`), and only then moves the ticket (`enterHumanReview`, split out of `concludeVerification` so the ticket stays unclaimable and the wizard unopenable while the agent reads).

The verdict stays human. The digest is opinion wearing a chip: the wizard renders it pre-filled — walkthrough and risks on the Recap step, acMap notes on the Manual Walkthrough items — every finding marked agent-generated, never mixed into chrome drawn from live data. Freshness is the digest's own (AC-43): the artifact row's worktree-HEAD stamp against the live branch tip, the same demo-fresh mechanics; a stale digest states its invalidation and stops pre-filling rather than annotating a diff the agent never saw.

Failure is a first-class ending (AC-42): a dead, hollow, or malformed digest is recorded on the audit trail (`review.digest`, status failed, reason) and the ticket still reaches Human Review — the wizard opens raw-diff wearing the flag. No retry: the digest is an accelerant, not evidence the verdict depends on, and the reviewer is already the fallback.

Considered: making the digest a workflow phase (rejected — ADR-0003 keeps verification-adjacent machinery orchestrator-side, and a phase would let workflow authors delete it or reorder it after the battery it must follow); retrying a failed digest like a phase death (rejected — it gates nothing, so a retry buys latency at the veto point for marginal value); LLM-summarizing gate results into the digest (rejected — gate results are structured data the chrome already renders; the digest's job is the diff, and Bounce Report doctrine forbids prose-first evidence).
